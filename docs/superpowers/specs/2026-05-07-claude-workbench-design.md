## 一、项目概述

### 1.1 项目背景

用户日常工作以文案输出为主，深度使用 Claude Code + OMC（oh-my-claudecode）构建 AI 工作站。当前使用终端运行 Claude Code，存在以下痛点：

1. **Team 模式状态黑盒**：无法了解各子系统运行状态
2. **内容预览割裂**：生成的文档/图片/页面需切换到其他程序查看
3. **模型切换不便**：需打开 cc-switch 或修改环境配置

### 1.2 解决方案

基于 Tauri 构建桌面客户端，集成 Claude Code 终端和内容预览功能，实现"一个界面完成所有工作"。

### 1.3 核心价值

- **无缝预览**：Claude Code 生成的文档、图片、网页直接在工作台预览，无需切换程序
- **状态可视化**：Team 任务进度和异常告警一目了然
- **高效集成**：全屏工作台，左导航 + 中终端 + 右预览，VS Code 风格布局

## 二、技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| **桌面框架** | Tauri 2.x | 轻量（<10MB）、安全、Rust 后端 |
| **前端框架** | React 18 + TypeScript | 成熟生态，组件丰富 |
| **UI 样式** | TailwindCSS | 快速构建美观界面 |
| **终端模拟** | xterm.js + node-pty | 嵌入式终端，支持 CLI 调用 |
| **状态管理** | Zustand | 轻量级状态管理 |
| **文件监听** | notify crate | Rust 文件系统监听 |
| **包管理** | pnpm | 快速节省空间 |

## 五、功能模块

### 5.1 终端模块

**核心功能**：
- 集成 xterm.js 终端模拟器
- 通过 node-pty 调用系统 `claude` CLI
- 支持 `claude code` 所有命令和交互
- 支持 Team 模式运行

**技术实现**：
- xterm.js：前端终端渲染
- node-pty：跨平台终端进程管理
- 方案 A（嵌入式调用 CLI）：Claude Code 升级基本无影响

### 5.2 预览模块

**支持的格式**（按优先级）：

| 优先级 | 格式 | 预览方式 |
|--------|------|----------|
| P0 | Markdown | 渲染为 HTML，带代码高亮 |
| P0 | Word (.docx) | 转换为 HTML 渲染 |
| P0 | PDF | PDF.js 渲染 |
| P0 | HTML | WebView 或 iframe 渲染 |
| P1 | 图片 (jpg/png/gif/svg) | 原生渲染 |
| P1 | PPT/PPTX | 转换为图片序列 |
| P1 | 思维导图 | 渲染为 SVG/HTML |
| P2 | Excel (.xlsx) | 表格渲染 |
| P2 | 视频 (mp4/webm) | 视频播放器 |
| P3 | 其他格式 | 文件信息展示 + 建议打开方式 |

### 5.3 Team 状态监控模块

**核心功能**：
- 任务进度可视化（进度条 + 百分比）
- 异常告警（高亮 + 详情）
- 点击展开任务详情
- 可查看单任务日志输出

**交互设计**：

```
状态栏默认：
┌─────────────────────────────────────────────────────────────┐
│ Team: [████████░░] 8/10 任务完成    ⚠ 1 个异常             │
└─────────────────────────────────────────────────────────────┘

点击展开后：
┌─────────────────────────────────────────────────────────────┐
│  Team 任务详情                                              │

│      └── 原因: 网络超时，重试中...                            │

```

### 5.4 文件输入模块

**两种输入方式**：

| 方式 | 触发机制 | 适用场景 |
|------|---------|----------|
| **Claude Code 输出推送** | Claude 生成文件后，通过 CLI 或文件事件通知工作台 | Claude Code 主动生成的内容 |
| **文件夹监听** | 工作台监控指定目录（如 `10-工作区/`），新文件自动出现在预览区 | 用户希望给 Claude Code 查看的内容 |
| **拖拽导入** | 用户手动把文件拖进工作台窗口 | 临时查看、非项目文件 |

### 5.5 设置模块

**配置项**：
- 监控文件夹路径配置
- 主题设置（深色/浅色）
- 字体大小
- 快捷键配置
- 窗口行为（启动时还原上次的窗口状态）
- 更新检查

## 七、扩展性设计

### 7.1 架构原则

- **前后端分离**：前端只负责 UI，后端负责业务逻辑
- **模块独立**：每个功能（终端、预览、Team）是独立组件
- **事件驱动**：模块间通过事件通信，不直接耦合
- **接口稳定**：Tauri Commands 接口保持稳定

### 7.2 后期迭代场景

| 场景 | 难易度 | 说明 |
|------|--------|------|
| 加新的预览格式 | 容易 | 只需在预览模块加一个新的 renderer 组件 |
| Team 加通知功能 | 容易 | 在 Team 模块加一个通知插件 |
| 加新的布局 | 容易 | 前端调整布局组件，后端不动 |
| 换终端方案 | 较易 | 终端模块独立封装，替换内部实现即可 |
| 升级 Claude Code 交互方式 | 较易 | 只需要改 Rust 后端的 CLI 调用层 |

### 7.3 技术债务管理

- 遵循语义化版本号（v1.0.0）
- 每次迭代记录变更日志
- 核心接口变更需 major version 升级
