# 开发记录：Phase 4 - 任务自动推进（闭环）

> 日期：2026-05-14
> 版本：v0.2.3-alpha
> 任务：Phase 4 - 自动推进与闭环

---

## 需求概述

实现 Agent 完成一个阶段后自动检测并推进到下一阶段：
1. 文件监听检测到阶段输出文件更新 → 提示用户确认完成
2. 用户确认后自动推进到下一阶段
3. 自动为下一阶段生成并注入新提示词
4. 全任务完成后总结报告

---

## 实现方案

### 1. 文件监听服务

新建 `src/services/fileWatcher.ts`：

- `FileWatcher` 类（单例）— 连接 Rust `notify` 文件监听器与前端业务逻辑
  - `startWatching(basePath)`：调用 Rust `start_file_watcher` Tauri command，监听 `file-change` 事件
  - `stopWatching()`：调用 Rust `stop_file_watcher`，清理事件监听
  - `setTask(task)`：设置当前任务，用于过滤相关文件变更
  - 回调：`onStageOutputChanged` — 当当前阶段的输出文件被修改时触发
  - 回调：`onAnyFileChanged` — 任何文件变更时触发

Rust 后端已具备的能力（复用）：
- `start_file_watcher(path)` — 递归监听目录
- `file-change` 事件 — 包含 `path` 和 `event_type`（create/modify/remove）

### 2. 自动推进逻辑

修改 `src/App.tsx`：

- **文件变更检测**：当 `file-change` 事件触发且匹配当前阶段输出文件时：
  - 添加聊天消息："检测到阶段输出文件已更新：{filename}，当前阶段是否已完成？输入'下一阶段'或'完成阶段'即可推进。"
  - 使用 `pendingAdvanceSuggestion` 状态防止重复提示

- **阶段完成时自动准备下一阶段**：
  - `handleCompleteStage` 完成后，查找下一个 running 阶段
  - 自动向聊天发送消息："阶段完成！已自动推进到「{nextStage.name}」。下一阶段提示词已就绪，切换到 Agent 运行标签页启动 Agent 即可自动注入。"

- **任务切换时清理状态**：
  - 切换任务时重置 `pendingAdvanceSuggestion`、文件监听、Agent 状态

- **Agent 退出时提示**：
  - Agent 会话结束时向聊天添加系统消息

### 3. 闭环流程

```
用户："帮我写个贵港供销社合作方案"
  ↓
Cospace：创建任务，启动需求确认阶段
  ↓
用户：启动 Agent
  ↓
Agent 运行 → 写入文件 → 文件监听检测到变更
  ↓
Cospace："检测到输出文件已更新，是否完成当前阶段？"
  ↓
用户："下一阶段"
  ↓
Cospace：完成阶段 → 自动推进 → 提示下一阶段已就绪
  ↓
用户：启动 Agent（下一阶段）
  ↓
... 循环直到所有阶段完成
```

## 已知限制

- 文件监听使用轮询间隔 1 秒（Rust 后端配置），可能有轻微延迟
- `pendingAdvanceSuggestion` 状态按任务 ID 不区分阶段，切换阶段后需手动重置
- 自动推进仍需用户通过聊天确认（"下一阶段"），未实现完全无人值守
