## 1. 产品定位

### 1.1 核心理念

**Cospace 不是 Agent，而是 Agent 驱动的工作平台。**

为方案/文案工作者提供一个集成化的工作台，固化个人工作方法，让 Agent 成为驱动工具的基础设施。

### 1.2 目标用户

- 经常撰写方案、文案、报告的知识工作者
- 需要管理多个并行项目的项目经理
- 希望沉淀工作方法、积累知识资产的创作者

### 1.3 与现有产品的差异

| 产品类型 | 特点 | Cospace 的差异 |
|---------|------|---------------|
| 通用 Agent 工具 | 直接聊天，对话驱动 | **工作台结构化**：任务-阶段-产出，有明确边界 |
| RPA/自动化工具 | 自动执行流程 | **人机协同**：人主导关键决策，Agent 执行辅助 |
| 文档协作工具 | 在线编辑文档 | **Agent 驱动**：Agent 理解上下文，主动调用工具 |
| 项目管理工具 | 看板、任务追踪 | **面向内容创作**：专为文案/方案场景设计 |

### 2.2 配置体系（混合模式）

#### 全局配置（用户级）

```
~/.cospace/config.json                  # 用户级全局配置
或
/工作区/.cospace/config.json            # 项目级全局配置

{
  "assets": {
    "templates": "~/notebook/templates",    # 文档模板库
    "data": "~/notebook/data",              # 数据文件
    "references": "~/notebook/references",   # 参考资料
    "cases": "~/notebook/cases"             # 历史案例
  },
  "workflows": "~/.cospace/workflows/",     # 工作流模板库
  "defaultWorkflow": "standard-4stage",   # 默认工作流
  "agent": {
    "type": "claude",
    "autoStart": true
  }
}
```

#### 任务级配置（覆盖全局）

```
/项目/任务A/.cospace/
├── task.json                       # 当前进度、Agent会话状态
├── workflow.md                     # 工作流定义（可复制全局或自定义）
└── context.jsonl                   # Agent对话历史（可选）
```

**配置继承规则**：
- 知识资产路径 → 全局配置（所有任务共享）
- 工作流定义 → 任务可覆盖（复制全局模板或自定义）
- 当前进度 → 任务级（必须跟随任务）
- Agent会话 → 任务级（每个任务独立）

## 3. 功能规格

### 3.1 任务管理（多任务并行）

#### 3.1.1 任务生命周期

```typescript
interface Task {
  id: string;
  name: string;
  path: string;                    // 工作目录绝对路径
  
  status: 'draft' | 'active' | 'paused' | 'completed' | 'archived';
  
  // 当前阶段
  currentStage: {
    index: number;                 // 当前阶段索引
    id: string;                    // 阶段标识
    name: string;                  // 阶段名称
  };
  
  // Agent会话
  agent: {
    sessionId: string;
    type: 'claude' | 'codex' | 'custom';
    status: 'idle' | 'thinking' | 'executing';
  };
  
  // 时间追踪
  createdAt: number;
  lastAccessed: number;
}
```

#### 3.1.2 任务操作

| 操作 | 说明 |
|------|------|
| 创建任务 | 选择工作流模板 → 输入名称 → 创建工作目录 → 初始化配置 |
| 启动任务 | 启动 Agent 会话 → 自动 cd 到任务目录 → 注入上下文 |
| 暂停任务 | 保留 Agent 会话状态，降低资源占用 |
| 恢复任务 | 恢复 Agent 会话，继续之前的工作 |
| 切换任务 | 切换焦点到指定任务，Agent 上下文自动切换 |
| 关闭任务 | 保存状态，关闭 Agent 会话 |
| 归档任务 | 移动到归档目录，保留完整元数据 |
| 删除任务 | 删除任务目录（可选择保留文档或完全删除） |

#### 3.1.3 多任务界面


### 3.2 工作流系统（配置驱动）

#### 3.2.1 工作流定义（workflow.md）

```markdown
# 贵港供销社方案工作流

## 元数据
- name: 贵港供销社方案
- type: proposal
- version: 1.0
- description: 面向供销社的供应链合作方案

## 阶段定义

### 阶段1：需求确认
- id: demand
- name: 需求确认
- description: 明确客户核心诉求和项目边界
- outputs:
    - name: 需求确认清单
      path: 01-需求确认/demand-checklist.md
      template: templates/demand-checklist.md
- agent_context: |
    当前阶段：需求确认
    任务：与贵港供销社确认项目需求
    你需要帮我：
    1. 梳理客户的核心诉求
    2. 明确项目边界和交付物
    3. 识别潜在风险和注意事项
    4. 产出需求确认清单

### 阶段2：框架构思
- id: framework
- name: 框架构思
- description: 确定方案整体结构和逻辑
- depends: demand
- outputs:
    - name: 方案框架
      path: 02-框架构思/framework.md
- agent_context: |
    当前阶段：框架构思
    上一阶段产出：{{demand.outputs[0].path}}
    任务：基于需求确认清单，构思方案的整体框架

### 阶段3：内容撰写
- id: draft
- name: 内容撰写
- description: 完成方案正文内容
- depends: framework
- outputs:
    - name: 方案初稿
      path: 03-内容撰写/draft.md

### 阶段4：审核定稿
- id: review
- name: 审核定稿
- description: 检查内容、格式，输出最终版本
- depends: draft
- outputs:
    - name: 最终方案
      path: 04-审核定稿/方案-最终版.docx
```

#### 3.2.2 工作流配置语法

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 阶段唯一标识（URL-friendly） |
| name | string | 阶段显示名称 |
| description | string | 阶段说明 |
| depends | string/string[] | 依赖的前置阶段 |
| outputs | OutputConfig[] | 阶段产出文档定义 |
| outputs.path | string | 产出文档相对路径（自动创建目录） |
| outputs.template | string | 模板路径（可选） |
| agent_context | string | Agent 系统提示词模板 |

### 3.3 Agent 驱动层

#### 3.3.1 上下文注入

工作台自动在每次对话前注入系统提示词：

```markdown
## 系统上下文

当前任务：{{task.name}}
任务类型：{{workflow.type}}
当前阶段：{{stage.name}}（{{stage.index}}/{{workflow.stages.length}}）
阶段目标：{{stage.description}}
工作目录：{{task.path}}

可用工具：
- /search [query] - 搜索知识库
- /template [name] - 引用模板
- /data [file] - 读取数据文件
- /skill [name] - 调用工作流技能
- /stage [next|prev|jump [id]] - 切换阶段

---

用户输入：
```

#### 3.3.2 工具调用机制

```typescript
// Agent 通过特殊语法调用工作台工具

// 1. 搜索知识库
/search query:供应链案例 category:模板 limit:5

// 2. 引用模板
/template load:方案框架-供应链

// 3. 读取数据
/data read:市场调研.xlsx sheet:2026

// 4. 切换阶段
/stage next

// 工作台解析指令 → 执行 → 返回结果给 Agent → Agent 继续处理
```

### 4.4 技术实现要点

#### 4.4.1 配置加载流程

```
1. 启动时加载全局配置
   ~/.cospace/config.json
   
2. 扫描任务目录
   遍历 /项目/*/.cospace/task.json
   
3. 加载任务配置
   读取 task.json + workflow.md
   
4. 合并配置
   任务级覆盖全局配置
```

#### 4.4.2 Agent 会话管理

```
创建任务
   ↓
创建工作目录 + .cospace/
   ↓
启动 Agent Session
   - 启动 PTY
   - cd 到任务目录
   - 启动 claude/codex
   ↓
注入系统提示词（首次对话）
   ↓
用户与 Agent 对话
```

#### 4.4.3 阶段切换

```
用户点击阶段N
   ↓
工作台：
   - 更新 task.json 中的 currentStage
   - 检查阶段目录是否存在，不存在则创建
   - 更新 Agent 系统提示词中的当前阶段
   - 可选：发送提示给 Agent "进入阶段N"
   ↓
Agent：继续对话，基于新阶段上下文
```

## 6. 技术实现建议

### 6.1 架构图

```
Cospace Desktop App
├── Frontend (React + TypeScript)
│   ├── TaskManager            # 任务管理
│   ├── WorkflowNavigator      # 工作流导航
│   ├── AgentChat              # Agent 对话界面
│   ├── AssetPanel             # 知识资产面板
│   └── Workspace              # 工作区布局
│
├── Backend (Rust + Tauri)
│   ├── TaskService            # 任务 CRUD
│   ├── WorkflowParser         # workflow.md 解析
│   ├── ConfigManager          # 配置管理
│   ├── AgentManager           # Agent 会话管理
│   └── FileWatcher            # 文件监听
│
└── External
    ├── Claude/Codex (PTY)    # Agent 进程
    └── Knowledge Assets      # 本地知识库
```

### 6.2 关键技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 配置格式 | Markdown (workflow.md) | 用户友好，易编辑，可版本控制 |
| 配置格式 | JSON (task.json) | 机器友好，性能高 |
| Agent 通信 | PTY/STDIO | 复用现有 Cospace 实现 |
| 状态管理 | 文件系统优先 | 数据持久化，用户可控 |
| 缓存 | 内存 + 文件 | 重启后可恢复 |

### A. 配置示例

#### 全局配置

```json
{
  "assets": {
    "templates": "~/notebook/templates",
    "data": "~/notebook/data",
    "references": "~/notebook/references",
    "cases": "~/notebook/cases"
  },
  "workflows": "~/.cospace/workflows",
  "defaultWorkflow": "standard-4stage",
  "agent": {
    "type": "claude",
    "autoStart": true
  }
}
```

#### 工作流模板（standard-4stage.md）

```markdown
# 标准方案工作流（4阶段）

## 元数据
- name: 标准方案
- type: proposal
- version: 1.0

## 阶段定义

### 阶段1：需求确认
- id: demand
- name: 需求确认
- description: 明确客户核心诉求
- outputs:
    - path: 01-需求确认/demand.md
- agent_context: 当前阶段：需求确认...

### 阶段2：框架构思
- id: framework
- name: 框架构思
- description: 确定方案整体结构
- depends: demand
- outputs:
    - path: 02-框架构思/framework.md
- agent_context: 当前阶段：框架构思...

### 阶段3：内容撰写
- id: draft
- name: 内容撰写
- description: 完成方案正文
- depends: framework
- outputs:
    - path: 03-内容撰写/draft.md
- agent_context: 当前阶段：内容撰写...

### 阶段4：审核定稿
- id: review
- name: 审核定稿
- description: 检查并输出最终版本
- depends: draft
- outputs:
    - path: 04-审核定稿/final.docx
- agent_context: 当前阶段：审核定稿...
```
